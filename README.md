# gogoclaw

`gogoclaw` is a Go implementation inspired by [OpenClaw](https://github.com/openclaw/openclaw) and [Nanobot](https://github.com/HKUDS/nanobot).

The goal is to make the system more modular, introduce clearer design boundaries, and reduce coupling so that components can be assembled, replaced, or detached quickly for Harness Engineering use cases.

The current repository is focused on:

- command entrypoints and basic CLI organization
- the onboarding bootstrap flow
- configuration schema and default config generation
- reserved repository layers for future `agent`, `memory`, `tools`, `session`, `gateway`, `channels`, `provider`, and related components

The repository is still in an early stage.

## Contributing

Before contributing, read `AGENTS.md` in the repository root.

Commit messages in this project are **recommended** to be generated by AI according to the rules in `AGENTS.md`:

1. Ask the AI to read `AGENTS.md` first.
2. Ask the AI to generate a commit message for the current change based on those rules.
3. Verify that the scope, predicate, and summary all match the convention before committing.

In particular:

- commit messages must follow `predicate(scope): summary`
- scopes should prefer the layer names defined in `AGENTS.md`
- if one change truly spans multiple primary layers, multiple scopes may be separated with commas

If a change affects code behavior, add or update the relevant tests and complete validation before committing.
